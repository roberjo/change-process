# Deployment Release Communication Templates

## 1. RFC Creation Notification

**Subject:** [RFC-XXXX] New Release RFC Created - [Release Name] - Review Required

**To:** QA Team, Business Partners, Support Teams, WE Leadership

**Template:**
```
Team,

A new RFC has been created for the upcoming release:

**RFC Details:**
- RFC ID: RFC-XXXX
- Release Name: [Release Name]
- Scheduled Implementation: [Date] at [Time] CST
- Implementation Window: [Duration]

**Action Required:**
- QA Team: Please provide testing signoff by [Date]
- Business Partners: Please provide business signoff by [Date]
- Support Teams: Review assigned tasks and confirm availability
- All Stakeholders: Review RFC documentation in [RFC System Link]

**Key Deliverables Needed:**
- Proof of testing artifacts
- Load testing results
- QA signoff documentation
- Business partner approval

**Timeline:**
- RFC Review Meeting: [Date/Time]
- Final Approval Deadline: Thursday [Date] by 5:00 PM CST
- Implementation Date: [Date/Time]

Please review the RFC and complete your assigned tasks by the specified deadlines.

Best regards,
[Release Manager Name]
```

```
**Attendees:**
- Release Manager: [Name]
- QA Lead: [Name]
- Business Partner Representative: [Name]

- WE Leadership: [Names]

**Materials:**
- RFC Document: [Link]

Please come prepared to discuss and approve/reject the RFC.
```

---

## Usage Guidelines

1. **Customize templates** with your specific system links, names, and processes
2. **Set up distribution lists** for consistent communication
3. **Use consistent RFC numbering** across all communications
4. **Include relevant links** to dashboards, documentation, and tracking systems
5. **Maintain communication logs** for post-implementation review
6. **Test emergency communication channels** before implementation
7. **Keep contact information current** in all templates
